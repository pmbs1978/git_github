Guia Completo de Git e GitHub

Um guia passo a passo para iniciar e gerir os seus projetos.

Introdução

Este tutorial explica de forma clara como criar um repositório Git no seu computador (local) e como conectá-lo a um repositório remoto no GitHub. Este é um fluxo de trabalho fundamental para qualquer programador.

A ideia central é que existem dois "locais":

  • Repositório Local: Uma pasta no seu computador onde o Git está a "observar" as alterações nos ficheiros.
  • Repositório Remoto: Um espaço nos servidores do GitHub onde o seu código fica guardado, servindo como backup e permitindo a colaboração.

Pré-requisitos

Antes de começar, garanta que tem o seguinte:

  1. Git Instalado: Se não tiver, pode instalá-lo no seu sistema. No Ubuntu/Debian: 
    sudo apt update && sudo apt install git
  2. Conta no GitHub: Uma conta gratuita em GitHub.com.
  3. Git Configurado: Configure o seu nome de utilizador e email no Git: 
    git config --global user.name "O Seu Nome"
git config --global user.email "o.seu.email@exemplo.com"

Parte 1: Começar um Projeto do Zero (Local)

Passo 1: Criar uma pasta para o projeto

Abra o terminal e crie uma nova pasta para o seu projeto, depois navegue para dentro dela. 

# Cria uma nova pasta chamada 'meu-primeiro-repo'
mkdir meu-primeiro-repo

# Entra na pasta que acabou de criar
cd meu-primeiro-repo

Passo 2: Iniciar o Git na pasta

Este comando "cria" o repositório local, adicionando uma pasta oculta .git para monitorizar as alterações. 

git init

Passo 3: Criar e Adicionar os seus primeiros ficheiros

Vamos criar um ficheiro README.md e adicioná-lo à "staging area" (área de preparação) do Git. 

# Cria um ficheiro README.md com um título dentro
echo "# O Meu Primeiro Repositório" > README.md

# Adiciona o ficheiro à 'staging area'
git add README.md

Passo 4: Fazer o "Commit" das alterações

Um "commit" é um ponto de salvaguarda no histórico do seu projeto. Vamos guardar as alterações que estão na staging area com uma mensagem descritiva. 

git commit -m "Commit inicial: Adiciona o ficheiro README"

Neste ponto, o seu repositório local está criado e já tem um registo no histórico.

Parte 2: Conectar o seu Projeto Local ao GitHub

Passo 1: Criar um novo repositório no GitHub

  1. Aceda à sua conta no GitHub.com.
  2. No canto superior direito, clique no ícone + e selecione "New repository".
  3. Dê um nome ao repositório (ex: meu-primeiro-repo).
  4. MUITO IMPORTANTE: Deixe as caixas "Add a README file", "Add .gitignore" e "Choose a license" desmarcadas. O repositório deve começar vazio para evitar conflitos.
  5. Clique em "Create repository".

Passo 2: Conectar o seu repositório local ao remoto

O GitHub irá mostrar-lhe uma página com comandos. Iremos usar os da secção "...or push an existing repository from the command line".

1. Adicionar o endereço remoto

Este comando define um "apelido" (origin) para o URL do seu repositório no GitHub. 

# Substitua o URL pelo URL do seu repositório
git remote add origin https://github.com/o-seu-username/meu-primeiro-repo.git

2. Renomear a branch principal para "main"

Esta é uma convenção moderna e uma boa prática. 

git branch -M main

3. Enviar o código (Push) para o GitHub

Este comando envia os seus commits locais para o repositório remoto no GitHub. 

git push -u origin main

Poderá ser pedido para fazer login com as suas credenciais do GitHub no terminal para autorizar esta ação.

Alternativa: Clonar um Repositório (git clone)

Este é o cenário mais comum quando quer contribuir para um projeto que já existe ou continuar a trabalhar no seu projeto noutro computador.

O comando git clone automatiza o processo: descarrega o projeto, cria o repositório local e configura a ligação ao remoto (origin) automaticamente.

Passo 1: Copiar o URL do Repositório Remoto

  1. Vá à página do repositório no GitHub que deseja clonar.
  2. Clique no botão verde <> Code.
  3. Certifique-se que a aba HTTPS está selecionada e copie o URL.

Passo 2: Clonar o Repositório

Abra o terminal na pasta onde quer que o projeto seja guardado. 

# Cole o URL que copiou do GitHub
git clone https://github.com/nome-do-dono/nome-do-repo.git

O Git irá criar uma nova pasta com o nome do repositório, descarregar todos os ficheiros e todo o histórico de commits. Já está pronto a trabalhar!

Bónus: Publicar com GitHub Pages

O GitHub Pages é um serviço gratuito que permite hospedar um site simples diretamente a partir do seu repositório. É perfeito para portfólios, páginas de projetos ou documentação.

Pré-requisito: Para que funcione, o seu repositório deve ter um ficheiro chamado index.html na raiz do projeto.

Passo 1: Ir às Definições

Na página do seu repositório no GitHub, clique na aba "Settings".

Passo 2: Secção "Pages"

No menu lateral esquerdo, clique em "Pages".

Passo 3: Configurar a Fonte

Na secção "Build and deployment", em "Source", selecione "Deploy from a branch".

Depois, em "Branch", selecione a branch main e a pasta /(root). Clique em "Save".

Passo 4: Visitar a Página

O GitHub irá demorar um ou dois minutos a publicar o seu site. Após esse tempo, a página irá recarregar e mostrar-lhe um link com o endereço da sua nova página, algo como: 

https://o-seu-username.github.io/nome-do-repo/

Parabéns! O seu projeto está agora online.

Ver o Histórico de Commits (git log)

Esta é uma das operações mais úteis no Git. O comando git log tem várias opções para formatar e filtrar a saída.

Formatos Populares 

# Ver cada commit numa única linha (muito útil)
git log --oneline

# Ver um gráfico de branches
git log --graph --oneline

# Ver estatísticas dos ficheiros alterados
git log --stat

Filtrar a Lista 

# Ver os últimos 3 commits
git log -n 3

# Ver commits de um autor específico
git log --author="Pedro Santos"

# Ver o histórico de alterações de um ficheiro
git log -- src/database.js

Voltar a um Commit Anterior (CLI)

É crucial entender as diferentes formas de o fazer. A escolha depende do seu objetivo.

1. Apenas para Inspeção: git checkout

Para "viajar no tempo" e ver como o projeto estava num ponto específico, sem o risco de perder trabalho. Para voltar ao presente, use git checkout main. 

git checkout 3b4c1a9

2. Reverter um Commit Público: git revert

Se o commit já foi partilhado (git push), esta é a forma mais segura. Em vez de apagar, cria um novo commit que desfaz as alterações, preservando o histórico. 

git revert f3b2c1a

3. Desfazer Commits Locais: git reset

git reset move o ponteiro do seu branch, reescrevendo o histórico. Use com muito cuidado e apenas em commits que ainda não partilhou.

AVISO: A opção --hard apaga permanentemente todas as alterações. Use apenas se tiver a certeza absoluta de que quer descartar o trabalho. 
# Apaga permanentemente o trabalho feito após o commit f3b2c1a
git reset --hard f3b2c1a

Alternativa: Usando a Interface do GitHub

A interface do site é ótima para visualizar o histórico e fazer reversões de forma segura, sem precisar de memorizar comandos.

Visualizar o Histórico no Site

  1. Na página do seu repositório, clique no link que diz "commits" (acima da lista de ficheiros).
  2. Pode clicar em cada commit para ver as alterações exatas.

Reverter um Commit no Site

Esta ação faz o equivalente a um git revert, preservando o histórico.

  1. Navegue até à lista de commits e clique no que deseja reverter.
  2. Na página do commit, clique no botão com três pontinhos (...) e selecione "Revert".
  3. O GitHub irá guiá-lo para criar um Pull Request (PR) com a reversão.
  4. Faça o merge do Pull Request para aplicar a alteração no branch principal.

Resumo do Fluxo de Trabalho Diário

Após a configuração inicial, o seu processo diário será:

  1. Sincronize com o remoto (importante em equipa): 
    git pull
  2. Trabalhe nos seus ficheiros (crie, edite, apague).
  3. Adicione as alterações à staging area: 
    git add .
  4. Faça o commit das alterações com uma mensagem: 
    git commit -m "Mensagem descritiva do que fiz"
  5. Envie as alterações para o GitHub: 
    git push
